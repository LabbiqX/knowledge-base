## TL;DR
AWS announced "frontier agents" — a new class of AI systems that can work autonomously for hours or days without human intervention. Three specialized agents were introduced: Kiro (software development), AWS Security Agent (application security), and AWS DevOps Agent (IT operations). Unlike existing AI coding assistants, these agents maintain persistent memory across sessions and can spawn multiple instances to work on different parts of a problem simultaneously.

---

## Key Points
- Frontier agents differ from existing tools by maintaining persistent memory across sessions and learning from an organization's codebase, docs, and team communications
- Kiro autonomous agent connects to GitHub, Jira, Slack; accepts task assignments and works independently
- AWS Security Agent transforms penetration testing from weeks-long manual process to hours
- AWS DevOps Agent diagnosed a complex network issue in 15 minutes (vs hours for experienced engineers)
- Agents can spawn 10 versions of themselves to work on different problem aspects simultaneously
- All learnings are logged and visible; teams can redact specific knowledge ("disconnect neurons")
- Agents never commit code directly to production — human responsibility remains
- AWS positions this against Google and Microsoft, leveraging 20 years of cloud infrastructure knowledge
