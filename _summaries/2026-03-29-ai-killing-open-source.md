## TL;DR
AI isn't killing open source, but it's actively undermining the social and economic assumptions projects have relied on for decades. AI slop (low-quality AI-generated contributions) is flooding maintainers. The feared problem — developers not contributing because AI makes code disposable — didn't materialize. Instead, the opposite happened: people submit AI-generated code they can't maintain or defend.

---

## Key Points
- AWS Head of Open Source Strategy Stormy Peters warns AI is undermining open source's foundational assumptions
- AI slop bug reports are fouling up security work (Daniel Stenberg/cURL example)
- The feared contribution drought didn't happen — instead, developers flood projects with AI-generated code
- Problem: contributors can't maintain or defend AI-generated code they submit
- "It's really quick, so I should contribute it" — but the follow-through is missing
- Maintainers face both quantity overload and quality degradation

## 🎯 Anahtar Çıkarım
> AI isn't killing open source — it's drowning it in unmaintainable contributions. The real threat isn't that developers won't contribute, but that they contribute code they don't understand and can't defend.
