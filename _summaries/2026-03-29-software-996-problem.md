## TL;DR
The author argues that AI-assisted coding replicates China's "996" overwork culture in silicon — more code output without better outcomes. GitClear analysis of 153M lines shows code churn spiking, more copy-paste, and less refactoring. AI helps code 55% faster but doesn't help build better. The "996 trap transferred to machines" assumes the constraint is characters typed, when it's actually clarity of thought. Every line of code is a liability, not an asset.

---

## Key Points
- "Code churn" (lines changed/thrown away within 2 weeks) spiking with AI usage
- AI makes coding 55% faster but increases bugs and reduces understanding
- "The real risk of AI isn't that it writes code, but that it encourages us to write too much code"
- Less code is better — every line must be secured, debugged, maintained, refactored
- Developers reviewing AI avalanche of PRs have no "slack" for innovation
- Solution: use AI to buy back time for human parts (problem framing, ruthless editing, owning blast radius)
- "The most valuable skill becomes deleting code" when AI makes writing it nearly free
- Celebrate "negative code" commits that remove complexity
