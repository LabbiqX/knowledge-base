## TL;DR
Microsoft created a new "engineering quality head" role and handed it to Charlie Bell (former security division head), less than a year after Nadella said 30% of Microsoft's code is AI-written. Code churn has roughly doubled since AI coding tools became widespread, developers miss 40% more bugs reviewing AI code, and Windows 11 has had serious reliability issues. Only 3.3% of Microsoft 365 users pay for Copilot.

---

## Key Points
- Nadella: 20-30% of Microsoft code is "written by software"; CTO predicts 95% by 2030
- GitClear research: code churn doubled after AI coding tools became widespread
- Microsoft's own research: 40% more bugs missed when reviewing AI-generated code
- Windows 11: security update broke boot, patches broke shutdown, emergency out-of-band fixes
- "Swarming" initiative: engineers pulled off features to fix reliability, expected to last months
- Charlie Bell moves from security EVP to individual contributor focused on quality
- Copilot adoption: only 3.3% of M365/O365 users pay for it
